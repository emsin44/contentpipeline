# Deployment, SEO & Performance

## Hosting: Cloudflare Workers

**Why:** Unlimited bandwidth, 300+ edge locations, <5ms cold starts, free SSL, git-based deploys, DNS in same dashboard.

### Config

```javascript
// astro.config.mjs
export default defineConfig({
  site: 'https://indiaesque.in',
  trailingSlash: 'always',
  output: 'server',
  adapter: cloudflare({ mode: 'advanced', imageService: 'compile' }),
  build: { format: 'directory' },
});
```

```toml
# wrangler.toml
name = "indiaesque"
compatibility_date = "2024-12-01"
compatibility_flags = ["nodejs_compat"]
main = "./dist/_worker.js/index.js"
[assets]
directory = "./dist"
```

### Deployment Flow

```
Push to main → Cloudflare builds → npm run build → wrangler deploy → Live (< 60s)
```

Build root directory: `india-experiences`

### Environment Variables

Set in Cloudflare dashboard (Workers & Pages → Settings → Variables):

| Variable | Used by |
|----------|---------|
| `VIATOR_API_KEY` | `viator.ts` |
| `GYG_PARTNER_ID` | `GygWidget.astro` |

### Domain

- `indiaesque.in` — DNS managed in Cloudflare
- `www.indiaesque.in` → 301 redirect via Cloudflare Redirect Rule
- DNS: `www` CNAME → `indiaesque.in` (proxied)

## SEO

### Title Tag

Formula: `{Page Topic} — {Site Name}` (under 60 chars, keyword first, year for time-sensitive)

Deduplication in `BaseLayout.astro`:
```javascript
const fullTitle = title.includes(siteName) ? title : `${title} | ${siteName}`;
```

### Meta Description

Formula: `{Direct answer}. {Supporting detail}. {Practical info: prices, duration}.`
120–155 characters, include keyword + specific data point.

### Schema URLs

All schema uses `https://indiaesque.in` consistently.

### CTA Alignment

City pages use editorial tone (not travel agency):
```html
<h2>DISCOVER MORE OF DELHI</h2>
<p>Explore our curated guides and hidden gems</p>
<a href="/journal/">BROWSE GUIDES</a>
```

## Sitemap

Auto-generated by `@astrojs/sitemap`.

- `https://indiaesque.in/sitemap-index.xml` — Master index
- `https://indiaesque.in/sitemap-0.xml` — All pages (~530 URLs)

**Critical:** Sitemap files excluded from Worker routing in `astro.config.mjs` so they're served as static XML:
```javascript
routes: { extend: { exclude: [
  { pattern: '/sitemap-index.xml' },
  { pattern: '/sitemap-0.xml' },
  { pattern: '/sitemap-*.xml' },
] } }
```

Submit to Google Search Console: `https://indiaesque.in/sitemap-index.xml`

## AI Crawler Optimization

### robots.txt

Allows all crawlers including GPTBot, ChatGPT-User, PerplexityBot, ClaudeBot, Google-Extended. Rate-limits AhrefsBot and SemrushBot.

### llms.txt

At site root — helps AI systems understand site structure. Lists content patterns, top cities, categories.

### Content Structure for AI

1. Direct answer in first 50 words
2. Clear heading hierarchy (h1 → h2 → h3)
3. Specific data in plain text (prices, durations)
4. No content gating (popups, sign-up walls)

## Performance

**Target: PageSpeed 90+ on mobile.**

### Requirements

- Total page weight < 500KB (above fold)
- FCP < 1.5s, LCP < 2.5s, CLS < 0.1
- Zero render-blocking JS
- Images: WebP, width/height attributes, `loading="lazy"` below fold
- Fonts: self-hosted, `font-display: swap`
- Compression: Brotli/Gzip via CDN
- Cache: static assets 1 year, HTML 1 hour

### Image Strategy

| Type | Max Width | Max Size | Format |
|------|-----------|----------|--------|
| Hero | 1200px | <100KB | WebP |
| Inline | 800px | <50KB | WebP |

Always include `width`, `height`, descriptive `alt` text.
